About Javinizer-js

Usage Instructions

Scrapers

Javinizer-JS supports multiple metadata scrapers that fetch information from various sources.

Manual Interaction Scrapers

Some scrapers require manual user interaction to bypass security measures:

  • javlibrary - Requires solving Cloudflare challenge and accepting adult agreement in the browser window that opens. Do not close the browser window.

FlareSolverr Required Scrapers

Scrapers ending with -fs require FlareSolverr to be installed and running:

  • javlibrary-fs

To use these scrapers, install FlareSolverr in a Docker Container:
https://github.com/FlareSolverr/FlareSolverr

Operation Modes

Scrape Mode

In Scrape mode, Javinizer-JS scans your library for video files, extracts their codes, and fetches metadata from configured scrapers. Results are saved as JSON files in data/scrape/ directory. You can then review, edit, and save them to your library with proper folder structure and NFO files.

Edit Mode

In Edit mode, Javinizer-JS loads existing NFO files from your library, allowing you to review and modify metadata. Changes are saved directly to the NFO files in your library folders.

Actor Management

Javinizer-JS can automatically scrape and cache actor data during movie save operations. Actor thumbnails and metadata are stored in data/actors/ by default, or in a custom path if configured. This allows for consistent actor information across your library.

Configuration

Visit the Configuration page to set up your library path, enable scrapers, configure actor scraping, and customize other settings. Changes are saved automatically and take effect immediately.
